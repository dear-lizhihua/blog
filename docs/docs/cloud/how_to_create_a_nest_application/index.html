如何创建一个 Nest 应用程序

参考链接#

  • Nest - First steps
  • Nest - Module

前置准备#

全局安装 Nest#

$ npm install -g @nestjs/cli

通过 Nest CLI 创建一个项目#

$ nest new learn-nest$ cd learn-nest

提示 Which package manager would you like to use? (Use arrow keys)，选择 yarn。

分析项目结构#

package.json 文件内容#

把 package.json 中无关的 Jest, ESLint, Prettier 删除，得到如下内容：

{  "name": "learn-nest",  "private": true,  "scripts": {    "build": "nest build",    "start": "nest start",    "start:dev": "nest start --watch",    "start:debug": "nest start --debug --watch",    "start:prod": "node dist/main"  },  "dependencies": {    "@nestjs/common": "^7.6.15",    "@nestjs/core": "^7.6.15",    "@nestjs/platform-express": "^7.6.15",    "reflect-metadata": "^0.1.13",    "rxjs": "^6.6.6"  },  "devDependencies": {    "@nestjs/cli": "^7.6.0",    "@nestjs/schematics": "^7.3.0",    "@types/express": "^4.17.11",    "@types/node": "^14.14.36",    "ts-loader": "^8.0.18",    "ts-node": "^9.1.1",    "tsconfig-paths": "^3.9.0",    "typescript": "^4.2.3"  }}

项目结构#

把无关的 README.md, .gitignore, .prettierrc, Jest 文件删除，得到如下项目结构：

$ tree.├── nest-cli.json├── package.json├── src│   ├── app.controller.ts│   ├── app.module.ts│   ├── app.service.ts│   └── main.ts├── tsconfig.build.json└── tsconfig.json

分析 package.json#

分析 scripts#

scripts说明
build构建一个 nest 应用程序
start启动一个 nest 应用程序
start:dev开发模式启动一个 nest 应用程序，此模式下修改文件触发重新编译和部署
start:debug调试模式启动一个 nest 应用程序，此模式在开发模式的基础上，增加调试的能力
start:prod生产模式启动一个 nest 应用程序，运行 dist/main.js

分析 dependencies#

package说明
@nestjs/common暴露出一些 decorator，例如项目中引用的 Module, Controller, Get, Injectable
@nestjs/core暴露 NestFactory.create()，用于创建一个应用程序。
@nestjs/platform-expressNest 文档 中提到，Nest 默认使用 Express
@nestjs/cliNest 应用程序的 CLI 工具。除了创建项目用到的 nest new 命令，nest start, nest build 命令也是通过此 package 来管理
@nestjs/schematicsNest 架构元素生成(architecture element generation)，基于 Angular schematics 理念

分析文件#

Nest 通过如下四个文件来运行应用程序：

  • src/main.ts
  • src/app.module.ts
  • src/app.controller.ts
  • src/app.service.ts

src/main.ts#

import { NestFactory } from '@nestjs/core';import { AppModule } from './app.module';
async function bootstrap() {  const app = await NestFactory.create(AppModule);  await app.listen(3000);}bootstrap();

@nestjs/core 中暴露出一个 NestFactory.create()，用于创建一个 Nest 应用程序。

然后此应用程序监听 TCP 3000 端口。

src/app.module.ts#

Nest - Module 文档中提到几个概念：

  1. 模块是用 @Module() decorator 注释的 class。@Module() decorator 向 Nest 提供用来组织应用程序结构的元数据(metadata)。
  2. 每个应用程序至少有一个模块，即一个根模块。根模块是 Nest 用于构建应用程序图(application graph)的起点。
  3. 应用程序图是一个用于解析 module(模块) 和 provider(提供者) 关系和依赖关系的内部数据结构。强烈推荐使用模块作为组织组件的有效方式。因此，对于大多数应用程序，最终的架构将采用多个模块，每个模块封装一组密切相关的功能。
import { Module } from '@nestjs/common';import { AppController } from './app.controller';import { AppService } from './app.service';
@Module({  imports: [],  controllers: [AppController],  providers: [AppService],})export class AppModule {}

这里是把用户编写的 Controller 和 Service 集成到 AppModule class 中，再传给 NestFactory.create() 初始化应用程序。

也就是说，应用程序的业务逻辑，最终会传给 Nest 框架。

metadata 对象#

key说明
controllers定义在此模块中的 controller，用于实例化。
providers每个 provider 都通过 Nest injector 进行实例化，并且可以在此模块中共享。
imports一个列表。导出给模块中的 provider，provider 用来导入。
exports一个 providers 子集。当前模块提供，并且用在导入此模块的其他模块中。

src/app.controller.ts#

import { Controller, Get } from '@nestjs/common';import { AppService } from './app.service';
@Controller()export class AppController {  constructor(private readonly appService: AppService) {}
  @Get()  getHello(): string {    return this.appService.getHello();  }}

这里是根模块的 controller。

在 constructor 中，可以看到将 AppService class 存放在 AppController 实例的 appService 属性上面。这样我们可以在 AppController 实例的 getHello 方法中，通过 this.appService 拿到 AppService 实例。

这里的 getHello 方法，被 Get decorator 包装过，所以通过 GET / 我们就可以进入到 getHello 方法中。

在 getHello 方法，我们调用 this.appService.getHello()，拿到 Hello World! 字符串，返回给 Nest 框架，最终输出在 HTTP 报文中。

src/app.service.ts#

import { Injectable } from '@nestjs/common';
@Injectable()export class AppService {  getHello(): string {    return 'Hello World!';  }}

这里是一个通过 Injectable 注入到根模块的 AppService。
